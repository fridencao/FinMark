## Design Context

### Users

**Primary**: Marketing managers, operations staff, and customer service managers at banks, securities firms, and other financial institutions.

**Context**: Users work at enterprise desks, making data-driven marketing decisions. They need clarity and confidence — not delight or playfulness. They switch between this tool and spreadsheets, reports, and meetings. Speed matters. Cognitive load must be minimized.

**Job to be done**: Orchestrate multi-agent AI workflows to generate marketing campaigns, evaluate results, and manage strategies — all within a compliant, auditable environment.

---

### Brand Personality

**Voice**: Professional, direct, precise. No marketing fluff. Technical but accessible.

**Tone**: Authoritative and trustworthy — like a senior consultant, not a friendly chatbot. Confidence without arrogance.

**3-word personality**: **Professional · Intelligent · Authoritative**

**Emotional goal**: Users should feel *in control* and *confident* when using this tool. The interface should reduce anxiety about AI decisions by providing transparency, structure, and clear feedback.

---

### Aesthetic Direction

**Visual tone**: Clean enterprise SaaS with premium polish — Linear meets Bloomberg Terminal. Data-dense where needed, but never cluttered. Generous whitespace in hero/input areas, compact but scannable in tables and dashboards.

**References**:
- Linear (app.linear.io) — clean hierarchy, restrained palette, professional confidence
- Stripe Dashboard — clear data presentation, subtle depth, trustworthy feel
- Bloomberg Terminal — information density done right

**Anti-references**:
- Consumer apps (no playful illustrations, no cartoon mascots)
- Startups with gradient-heavy hero sections
- Anything that feels "AI startup hype" (no sparkle animations, no robot mascots)

**Theme**: Light-first with full dark mode support. Indigo/neutral palette. Data visualization in blues with purposeful accent colors for status (emerald for success, amber for warning, red for error).

---

### Design Principles

1. **Clarity over cleverness** — Every UI element earns its place. If it doesn't inform or act, it doesn't exist. No decorative complexity.

2. **Confidence through transparency** — Show AI reasoning, not just results. Give users enough context to trust and override AI decisions. Full audit trail visible.

3. **Information density with breathing room** — Dense data in tables and dashboards, but generous padding in input flows and workflow stages. Never feels cramped.

4. **Predictable patterns, purposeful motion** — Use Motion/react for meaningful transitions only (state changes, panel reveals). No gratuitous animation. Respect `prefers-reduced-motion`.

5. **Professional polish in every detail** — Consistent 0.625rem border-radius, Geist Variable typography, precise spacing scale. The small things build trust.
