## 1. 产品概述

### 1.1 产品背景

FinMark AI 是基于多智能体协作的金融智能营销平台，旨在帮助金融机构实现客户的精细化运营。产品依托大模型能力，覆盖客户洞察、客群生成、内容创作、策略制定及效果评估全链路营销流程。

### 1.2 产品定位

企业级 B2B 金融营销科技产品，面向银行、券商等金融机构，提供私有化部署的智能营销解决方案。

### 1.3 目标用户

- **客户群体**：大型国有银行/券商、中小型金融机构
- **决策部门**：业务部门（营销部门、运营部门）
- **使用人员**：营销经理、运营人员、客服经理

### 1.4 产品价值

- 提升营销效率：自动化生成营销方案，减少人工干预
- 精准客群运营：基于数据驱动，实现客户分层精细运营
- 合规风控：内置金融合规规则，自动审核营销内容
- 效果可量化：实时监控营销效果，ROI 分析与优化建议

## 3. 核心功能模块

### 3.1 智能体协同流程

```
营销目标 → 洞察智能体 → 客群智能体 → 内容智能体 → 合规智能体 → 策略智能体 → 评估智能体
                                      ↓
                                A/B 测试版本选择
```

### 3.2 洞察智能体

**功能定位**：基于营销目标，识别潜在目标客群，生成客户标签

**输入**：
- 客户标签画像数据（来自大数据平台）
- 营销目标（如：提升基金销售、挽回流失客户）

**输出**：
1. 结构化 JSON（下游消费）
2. 人类可读报告（业务查看）

**输出 JSON 结构**：
```json
{
  "insight_id": "xxx",
  "target_goal": "新发基金推广",
  "generated_tags": ["ESG兴趣", "高频交易客户"],
  "potential_audience": [
    {
      "tag": "基金持有者",
      "count": 5000,
      "priority": "high",
      "reason": "历史持有基金，有投资经验"
    }
  ],
  "insights": [
    {
      "type": "demand",
      "description": "部分客户对ESG主题基金有意向",
      "confidence": 0.85
    }
  ],
  "recommendations": [
    "建议优先触达基金持有者，转化周期短",
    "高风险偏好客户需配合适当的风险提示"
  ]
}
```

**核心能力**：
- 读取行方现有客户标签
- 基于分析生成新标签（先生成再调用）
- 输出客户 ID 维度，展示由哪些标签组合筛选而出
- 客群规模超过 10 万时提醒客户指定上限

---

### 3.3 客群智能体

**功能定位**：将洞察结论转化为可执行的客群筛选条件，生成客户 ID 列表

**输入**：
- 洞察智能体输出
- 营销目标
- 约束条件（客群规模上限，默认 10 万）

**输出**：
1. 结构化 JSON（下游消费）
2. 人类可读报告

**输出 JSON 结构**：
```json
{
  "segment_id": "xxx",
  "source_insight_id": "xxx",
  "filters": [
    {"tag": "基金持有者", "operator": "in", "values": ["是"]},
    {"tag": "风险等级", "operator": ">=", "values": ["R4"]}
  ],
  "customer_count": 8500,
  "profile": {
    "avg_age": 35,
    "avg_aum": 500000,
    "top_tags": [...]
  }
}
```

**核心能力**：
- 自动生成筛选条件
- 业务人员确认后可修改
- 支持客群重命名
- 支持保存为可复用模板

---

### 3.4 内容智能体

**功能定位**：生成个性化的营销文案，针对不同客群生成不同版本

**输入**：
- 客群智能体输出
- 营销目标
- 营销渠道
- 品牌调性与合规要求

**输出**：
1. 结构化 JSON（下游消费）
2. 人类可读报告

**输出 JSON 结构**：
```json
{
  "content_id": "xxx",
  "segment_id": "xxx",
  "variants": [
    {
      "variant_id": "v1",
      "channel": "短信",
      "content": "尊敬的客户，您好！我行现推出...",
      "variables": {
        "customer_name": "{{name}}",
        "product_name": "ESG主题基金"
      },
      "length": 68,
      "tone": "正式"
    }
  ]
}
```

**核心能力**：
- 默认生成 3 组文案
- 支持业务人员指定文案风格
- 支持多种变量：客户姓名、产品名称、活动名称、权益信息、日期等
- 自动适配不同渠道的内容长度限制

---

### 3.5 合规智能体

**功能定位**：审查文案与策略的合规性，确保金融合规

**输入**：
- 生成的文案内容
- 客群风险等级分布
- 营销目标涉及的产品信息

**输出**：
1. 结构化 JSON（下游消费）
2. 人类可读报告

**输出 JSON 结构**：
```json
{
  "compliance_id": "xxx",
  "content_id": "xxx",
  "status": "passed",
  "warnings": [
    {
      "type": "risk_mismatch",
      "severity": "high",
      "description": "R3及以下客户推荐了R5高风险产品",
      "affected_count": 1200,
      "suggestion": "建议排除风险等级R3以下的客群"
    }
  ],
  "approved_variants": ["v1", "v3"],
  "rejected_variants": ["v2"]
}
```

**核心能力**：
- 内置金融合规标准
- 支持行方自定义合规规则
- 产品风险等级与客群风险等级匹配（对接产品画像数据）
- 禁语词库支持行方提供与系统内置，支持定期更新

---

### 3.6 策略智能体

**功能定位**：制定多渠道触达路径与预算分配

**输入**：
- 合规智能体审核通过的文案
- 客群画像与规模
- 营销目标与预算
- 渠道配置（行方支持的触达渠道）

**输出**：
1. 结构化 JSON（下游消费）
2. 人类可读报告

**输出 JSON 结构**：
```json
{
  "strategy_id": "xxx",
  "segments": [
    {
      "segment_id": "s1",
      "channel_sequence": [
        {
          "step": 1,
          "channel": "企微",
          "timing": "立即发送",
          "expected_reach_rate": 0.6,
          "content_variant": "v1"
        }
      ],
      "budget_allocation": 5000,
      "expected_roi": 3.2
    }
  ],
  "total_budget": 15000,
  "timeline": {...}
}
```

**核心能力**：
- 渠道可配置（实施时可增加新渠道）
- 智能体推荐优先级，业务可修改
- 预算默认按客群分配，也可按渠道分配，支持手动调整
- 支持定时任务、周期任务、事件任务、实时触发任务
- 自动生成 A/B 测试多个策略版本供选择

---

### 3.7 评估智能体

**功能定位**：实时监控营销效果，ROI 分析与优化建议

**输入**：
- 策略智能体输出的执行计划
- 实际执行数据
- 成本数据

**输出**：
1. 结构化 JSON（下游消费）
2. 人类可读报告

**输出 JSON 结构**：
```json
{
  "evaluation_id": "xxx",
  "strategy_id": "xxx",
  "overall_metrics": {
    "total_reach": 8500,
    "total_response": 1200,
    "total_conversion": 85,
    "total_cost": 15000,
    "roi": 2.8,
    "conversion_rate": 0.01
  },
  "anomaly_detection": [...],
  "optimization_suggestions": [...]
}
```

**核心能力**：
- 支持对接行方数据平台采集执行数据
- 指标由客户提前定义，系统引用
- 支持配置告警阈值与通知方式
- 支持历史数据对比与归因分析

## 4. 场景管理

### 4.1 场景模板

系统预置 4 个默认场景模板：

| 场景 | 分类 | 说明 |
|------|------|------|
| 流失挽回 | 挽回期 | 识别近30天资产下降超过30%的客户并进行挽回营销 |
| 新发基金推广 | 成长期 | 针对有理财经验且风险偏好为中高风险的客户推广新发ESG基金 |
| 信用卡分期提升 | 成长期 | 筛选有大额消费记录但未办理分期的客户，推送分期优惠券 |
| 个人养老金开户 | 获客期 | 针对符合开户条件且未开立养老金账户的代发工资客户进行推广 |

### 4.2 场景配置结构

```json
{
  "scenario_id": "xxx",
  "name": "场景名称",
  "description": "场景描述",
  "goal": "营销目标",
  "category": "获客期/成长期/成熟期/衰退期/挽回期",
  "insight_config": {
    "target_tags": ["目标标签"],
    "analysis_logic": "洞察逻辑描述"
  },
  "segment_config": {
    "criteria": "客群筛选条件"
  },
  "content_config": {
    "style": "文案风格",
    "channels": ["支持渠道"]
  },
  "strategy_config": {
    "path": "触达路径"
  }
}
```

### 4.3 场景分类

- 默认按客户生命周期划分：获客期、成长期、成熟期、衰退期、挽回期
- 支持行方自定义分类维度

### 4.4 AI 智能生成场景

- 支持通过自然语言描述自动生成场景配置
- 交互流程：用户输入描述 → AI 生成 → 业务人员确认/修改 → 生效

## 5. 策略引擎

### 5.1 策略原子库

**原子类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| 钩子 | 吸引客户注意的策略 | 高收益推荐、限时优惠 |
| 渠道 | 触达路径组合 | 企微+短信组合 |
| 内容 | 文案风格模板 | 温情关怀、专业推荐 |
| 风险 | 风控策略 | 风险等级匹配、合规校验 |

**核心能力**：
- 支持行方自定义策略原子
- 支持基于活动创建策略
- 支持绑定特定场景或设为通用
- 记录策略的使用场景
- 支持智能推荐 + 人工调整

### 5.2 A/B 测试

**支持测试类型**：
- 文案 A/B 测试
- 渠道组合测试
- 触达时间测试
- 人群细分测试

**核心能力**：
- 测试组使用默认策略，对照组使用大模型推荐最优策略
- 支持智能体生成多个版本供选择
- 流量分配支持：随机、百分比、特征分配
- 测试结束按活动周期判定
- 自动推荐获胜版本
- 支持多组测试对比

### 5.3 任务调度

**触发类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| 定时任务 | 指定具体时间执行 | 1月15日 10:00 发送 |
| 周期任务 | 按周期重复执行 | 每周一、三、五 9:00 |
| 事件任务 | 满足条件立即触发 | 客户首次购买后 24h 内 |

**任务配置项**：
- 任务名称
- 关联场景/策略
- 触发条件
- 执行时间
- 目标客群
- 渠道
- 重试策略
- 告警配置

## 6. 运营管理

### 6.1 效果仪表盘

**核心指标**：
- 触达指标：触达量、触达率、曝光量
- 响应指标：响应率、点击率、回复率
- 转化指标：转化率、ROI、客单价
- 成本指标：总成本、单客成本、渠道成本

**核心能力**：
- 支持按时间/渠道/场景维度筛选
- 支持自定义布局
- 支持自定义指标显示/隐藏

### 6.2 报表中心

**报表类型**：
- 活动报表：单次营销活动效果汇总
- 客群报表：客群分析与统计
- 内容报表：文案效果分析
- 渠道报表：渠道效果分析

**核心能力**：
- 自动生成日报、周报、月报
- 支持自定义报表周期
- 支持导出 Excel/PDF
- 支持定时自动推送
- 支持自定义报表

### 6.3 告警管理

**告警类型**：
- 指标告警：ROI低于阈值、转化率低于预期
- 任务告警：发送失败、重试耗尽
- 系统告警：服务不可用、数据延迟

**核心能力**：
- 支持配置指标阈值、触发条件、通知方式、通知人员
- 支持告警历史记录与处理流程
- 支持配置告警静默期

## 8. 部署要求

### 8.1 私有化部署

**基础设施**：
- 服务器：建议配置 8 核 CPU、32GB 内存、500GB SSD
- 操作系统：CentOS 7+ / Ubuntu 20.04+
- 数据库：MySQL 8.0+ / PostgreSQL 14+
- 中间件：Redis、Nginx

**部署方式**：
- 支持容器化部署（Docker/K8s）
- 提供一键部署脚本
- 提供系统监控工具与运维管理界面

### 8.2 模型配置

**模型接入**：
- 支持多种大模型（文心一言、通义千问、GLM 等）
- 支持 API 调用与行方私有部署模型
- 可配置 API 地址、密钥、模型版本、温度参数等

**模型管理**：
- 支持多模型切换
- 记录模型调用日志
- 监控模型调用量和响应时间
- 每个智能体可独立配置不同模型
- 支持降级策略（备用模型/本地规则引擎）

## 9. 非功能需求

### 9.1 性能要求

| 操作 | 响应时间 |
|------|----------|
| 页面加载 | < 3 秒 |
| 智能体生成 | < 30 秒（复杂分析 < 60 秒） |
| 数据查询 | < 5 秒 |
| 批量操作 | < 60 秒 |

- 支持多用户并发
- 单次客群筛选最大支持 10 万（超过提醒客户指定上限）

### 9.2 安全要求

**认证与授权**：
- 支持用户名密码、LDAP/AD 域控、短信验证码认证
- 基于角色的权限控制（RBAC）

**数据安全**：
- HTTPS 加密传输
- 敏感数据（手机号、身份证号）加密存储与脱敏展示
- 记录所有操作审计日志，支持查询与导出

### 9.3 可靠性要求

- 系统可用率：99.9% 以上
- 故障恢复：30 分钟内
- 数据备份：每日备份，保留 30 天

### 9.4 合规要求

- 数据不出网：所有数据本地化存储与处理
- 审计留痕：关键操作完整记录、可追溯
- 权限管控：敏感数据访问权限严格控制
- 合规智能体自动审核文案
- 敏感词库实时过滤
- 支持导出合规检查报告

## 10. 验收标准

### 10.1 功能验收

**智能体功能**：

| 智能体 | 验收标准 |
|--------|----------|
| 洞察智能体 | 能正确读取客户标签、能生成新标签、输出JSON+报告、客群ID维度输出、超过10万有提醒 |
| 客群智能体 | 能自动生成筛选条件、能手动修改、能保存模板、能重命名 |
| 内容智能体 | 能生成3组文案、能替换变量、能适配渠道长度限制 |
| 合规智能体 | 能审核文案、能审核策略、能对接产品画像、词库可更新 |
| 策略智能体 | 能编排多渠道、能生成AB测试版本、能手动调整预算、支持定时/周期/实时任务 |
| 评估智能体 | 能展示核心指标、能告警、能历史归因分析 |

**场景管理**：
- 4个预设场景可正常使用
- 可创建、编辑、删除场景
- 支持按生命周期分类、支持自定义分类
- 能通过自然语言生成场景配置

**策略引擎**：
- 支持自定义策略原子、支持按场景绑定、记录使用场景
- 支持多种测试类型、生成多版本、流量分配、推荐获胜版本
- 支持定时/周期/事件任务、暂停/恢复/终止、日志查看

**运营管理**：
- 多维度筛选、自定义布局、核心指标可视化
- 多种报表类型、支持导出、支持定时推送
- 多类型告警、阈值配置、通知方式、处理流程

### 10.2 系统集成验收

| 系统 | 验收标准 |
|------|----------|
| CRM系统 | 能读取客户标签、能输出客群名单 |
| 权益系统 | 能读取权益信息、能输出名单+权益 |
| 渠道系统 | 能读取渠道配置、能下发触达指令、能接收执行结果 |
| 大数据平台 | 能读取行为数据/产品画像、能回传分析结果 |

### 10.3 非功能需求验收

- 性能：页面加载<3s、智能体生成<30s/60s、数据查询<5s
- 安全：多种认证方式、RBAC权限、HTTPS传输、敏感数据脱敏、审计日志
- 可靠性：可用率99.9%、30分钟故障恢复、每日备份
- 合规：数据不出网、审计留痕、合规报告导出

### 10.4 部署验收

- 私有化部署：容器化部署正常、一键部署脚本正常执行
- 模型配置：多模型接入正常、切换正常、调用日志完整
- 运维管理：监控界面正常、告警通知正常、日志可查询
